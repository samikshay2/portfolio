The Boat Race

Project: Scratch Game

Course: Computer Science / Software Engineering (CSE)

Duration: Approximately 1.5 weeks (August 29 - September 8, 2017)

Objective: Create an original game with varying levels of difficulty and user interaction

Type of Project: Pair Programming (Samiksha Yelthimar and Derek Xu)

Link to Project (Game): The Boat Race Game

Link to Project Documentation: The Boat Race Documentation


In the Scratch Project, we had to create an original game of our choice. This was a two person project, and I was co-lead in the project. My partner for this project and I, thought that a boat-themed game could be very interesting if we added obstacles of varying difficulties. We also wanted to add multiple levels to this game, and give the player the choice to decide what level they want to start on. I focused strongly on the design, and making this game engaging through the use of various obstacles, such as sharks and oil spills. The objective of our game was to pass each level as fast as you can. If you hit the borders, or an obstacle, you will be prompted to start over from the beginning. It was difficult making all of the sprites to work as intented, especially working in the new Scratch environment, but we figured it out through trying over and over again. We created a Project Design Notebook that is linked under the section for project documentation. I learned various skills, including programming samples for computer science (Scratch Programming Environment), and creating a Project Design Notebook for engineering,

Snapshot Photobooth

Duration: Approximately 3 weeks (March 6 - March 21, 2018)

Objective: To write python script that does several manipulations to a large number of images in a given directory.

Type of Project: Pair Programming (Samiksha Yelthimar and Darlena Tran)

Link to Project (In GitHub): Snapshot PhotoBooth Code in GitHub

Link to Project (In Cloud9, under folder 1.4.7): Snapshot PhotoBooth Code in Cloud9

In the Image Manipulation, we had to create an image manipulation program that would run against a large number of files in a directory. This was a two person project, and I was co-lead in the project. My partner for this project and I, thought that a making the images all the same size, black and white, adding a border with a geometric accent, some symbols, and a nametag would make the images nice. We also wanted to give the client options for who they wanted to change the image - a girl or a boy. I focused strongly on the design, deciding how exactly to change the images. I also wrote much of the code for the different manipulations and making a python "for" loop that iterates through all images in a directory. It was difficult getting the "for" loop to correctly loop through all the images, but my partner and I referenced past assignments from class that ultimately helped us get through the technical difficulties. We created a Project Design Notebook that is linked above, under the section for project documentation. I learned various skills, including programming samples (in Python) for computer science, and creating a Project Design Notebook for engineering.
